# Code of Conduct

CodeDeck should be a useful and welcoming project for people with different backgrounds and levels of experience.

## Expected behavior

When participating in issues, discussions, pull requests or other project spaces:

- be respectful and constructive
- focus feedback on the code or idea, not the person
- explain technical decisions clearly
- accept that others may have different experience or priorities
- give new contributors enough context to participate
- respect privacy and do not publish personal information

## Unacceptable behavior

The following is not accepted:

- harassment, insults or personal attacks
- discriminatory or sexualized comments
- threats or intimidation
- deliberate disruption of project discussions
- publishing someone else's private information
- repeated bad-faith behavior after being asked to stop

## Reporting a problem

Report conduct problems privately to the repository maintainer through their GitHub profile. Do not open a public issue containing private details.

Include links, screenshots or other relevant context where possible. Reports will be reviewed in context and handled as fairly as possible.

## Enforcement

Project maintainers may edit or remove comments, close discussions, reject contributions or temporarily or permanently restrict participation when behavior harms the project or its contributors.

This code of conduct applies to all official CodeDeck project spaces.
